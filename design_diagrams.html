Before and After: Addition of the User Administrator Role

Before — No User Administrator Role

Three actor types exist: Anonymous visitors, regular logged-in users, and Club Executives (who gain elevated access by entering a club's executive code).

Diagram Description

In the original design, HoosLinked supports three categories of users. Anonymous visitors can browse the site without an account — they can view the home page, search for clubs, browse the Get Involved listing, see upcoming events, and read public announcements on club detail pages. However, they cannot interact with any club content or access any personalized features.

Once a user creates an account and logs in, they become a Regular Member. Members can manage their profile, view and join clubs, RSVP to events, send direct messages to other users, and participate in club forums — but only for clubs they have already joined. They can also enter an executive code to self-upgrade their membership to Club Executive status for a specific club.

A Club Executive retains all regular member capabilities and gains a set of club management tools scoped to the clubs they oversee: they can create events, post announcements (visible to everyone or members only), upload a club banner image, and access the executive dashboard which summarizes club membership and event statistics.

After — User Administrator Role Added

A fourth actor type is introduced: the User Administrator. This account is created exclusively through the Django admin panel and has a dedicated dashboard for managing user membership roles across all clubs. User Admins are redirected away from normal user pages and have no access to club content.

Key change: The User Administrator is the only role created outside the application UI — it requires a Django superuser to set is_user_admin = True on a user record in the Django admin panel. Once logged in, the User Admin is automatically redirected to /user-admin/ and cannot access profile pages, club pages, or messaging features.

Diagram Description

The updated design introduces a fourth user role: the User Administrator. The three existing roles — Anonymous Visitor, Regular Member, and Club Executive — remain identical in behavior to the original design. Anonymous users can still browse public content without logging in. Regular members can join clubs, participate in forums, RSVP to events, and message other users. Club Executives retain their club management capabilities including creating events, posting announcements, and uploading club images.

The new User Administrator role is a privileged, staff-only account created entirely outside of the normal application flow. Rather than signing up through the website, a User Admin account is provisioned by a superuser setting the is_user_admin flag directly in the Django admin panel. This deliberate separation ensures that no regular user can elevate themselves to administrator status. Once a User Admin logs in, the application automatically redirects them to a dedicated admin dashboard at /user-admin/, bypassing all standard user pages. From this dashboard, the administrator can view every non-admin user along with their club memberships, and can promote or demote any user between the Member and Executive roles for any club. The User Admin has no access to club content, forums, messaging, or events — their sole purpose is role governance across the platform.
